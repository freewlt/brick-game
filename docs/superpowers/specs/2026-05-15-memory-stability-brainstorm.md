# 运行内存不足闪退 Brainstorm

**Date:** 2026-05-15
**Status:** 已形成优化方案并执行
**Plan:** [docs/superpowers/plans/2026-05-15-memory-stability.md](../plans/2026-05-15-memory-stability.md)

---

## 背景

微信小游戏运行时频繁提示运行内存不足并闪退。当前项目是单 Canvas 全屏重绘，`GameScene` 中棋盘、槽位、粒子、音效都会在游戏过程中持续运行，因此优先排查“每帧分配”和“长时间运行资源不释放”。

## 目标

- 降低真机运行内存和显存占用
- 保留核心玩法、音效反馈和整体视觉风格
- 避免大范围重构，先做低风险、可回滚的稳定性修复
- 保证现有单测通过

## 关键问题

| 问题 | 位置 | 影响 |
|------|------|------|
| Canvas 按设备 DPR 原样放大 | `game.js` | DPR=3 设备会让画布物理像素扩大 9 倍，显存压力很高 |
| 棋盘逐格创建渐变 | `GameScene._drawBoard()` / `_drawSlot()` | 每帧为大量格子创建 `CanvasGradient`，长时间运行压力大 |
| 动画数组每帧 `filter()` | `GameScene.update()` | 每帧创建新数组，属于热路径分配 |
| 消除粒子过多 | `GameScene.update()` | 连续消除时粒子对象增长较快 |
| 木琴音每次创建短噪声 buffer | `audio.js::_marimba()` | BGM 和点击音会频繁创建 `AudioBuffer` |
| 音频节点播放后未主动断开 | `audio.js` | WebAudio 节点可能比预期更晚释放 |
| 通关/失败后 BGM 继续排队 | `GameScene.update()` | 结算前仍可能有 BGM 音符队列活动 |
| 非 BGM 音效延迟回调未统一管理 | `audio.js` / `GameScene.destroy()` | 离开游戏场景后旧音效 timer 仍可能创建节点 |
| 结算页全通关跳转 timer 未销毁 | `ResultScene` | 离开结算页后旧回调仍可能触发跳转并持有旧 scene |
| 渲染循环缺少异常兜底 | `game.js` | draw/update 一旦抛错，用户看到黑屏，体验上接近闪退 |

## 方案取舍

### 方案 A：DPR 封顶到 2

优点：
- 对内存最直接，DPR=3 设备像素量从 9x 降到 4x
- 对本项目卡通大色块画面影响较小

代价：
- 极高清屏上细节锐度会略降

结论：采用。稳定性收益大于轻微清晰度损失。

### 方案 B：保留所有渐变但做离屏缓存

优点：
- 视觉最接近原版
- 可把格子渲染变成 `drawImage`

代价：
- 需要管理离屏 canvas 与不同颜色、尺寸、状态缓存
- 代码改动更大，微信小游戏环境下缓存本身也占显存

结论：暂不采用，留作后续二阶段优化。

### 方案 C：渐变对象缓存复用

优点：
- 不改变棋盘格、车块、槽位的玻璃渐变视觉
- 能减少每帧重复创建 `CanvasGradient` 带来的 GC 压力

代价：
- 需要维护 `_gradCache` key，尺寸或布局变化时要清空缓存

结论：采用。

### 方案 D：关闭全部音效

优点：
- 最省内存

代价：
- 反馈明显变差，点击和消除体验损失大

结论：不采用。只移除噪声 buffer 和主动释放节点，保留主音效。

### 方案 E：降低粒子数量并封顶

优点：
- 低风险，减少连续消除时对象数量

代价：
- 消除效果略收敛

结论：采用。

## 最终方案

1. Canvas DPR 封顶到 `2`
2. 棋盘格、车块、槽位渐变改为缓存复用，不改视觉效果
3. 动画数组从 `filter()` 改为原地压缩清理
4. 消除粒子从每次 `18` 个降到最多 `8` 个，总量封顶 `64`
5. 移除木琴音短噪声 buffer
6. WebAudio 节点播放结束后主动 `disconnect()`
7. 关卡胜利/失败后立即停止 BGM
8. 场景切换统一调用 `destroy()`，清理旧场景动画、渐变缓存、BGM 和短音效 timer
9. `ResultScene` 销毁时清理全通关自动跳转 timer
10. 渲染循环增加异常兜底，避免单帧错误导致黑屏

## 后续观察点

- 如果仍然闪退：把 DPR 从 `2` 进一步降到 `1.5`
- 如果用户反馈模糊：试 `2.5` 或仅对低内存设备使用 `2`
- 如果渐变缓存仍有压力：再评估离屏 canvas 缓存车块图块
- 如果音频仍疑似占内存：提供设置项关闭 BGM，保留点击/消除短音效
