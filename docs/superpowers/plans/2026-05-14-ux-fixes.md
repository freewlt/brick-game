# UX Fixes: Level Progress / Leaderboard Cache / Friend Rank Analysis

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Fix three UX issues: (1) after winning a level the retry button incorrectly restarts from level 1; (2) document why friend leaderboard data is unavailable for individual developers and list alternatives; (3) cache leaderboard data on the Game object so re-entering the leaderboard shows stale data instantly while silently refreshing.

**Architecture:** All fixes are isolated to existing files. The level-progress fix is a one-line change in `ResultScene.js`. The leaderboard cache is added to the `Game` object in `game.js` and consumed in `LeaderboardScene.js`. The friend-rank analysis is documentation only (no code change).

**Tech Stack:** WeChat Mini Game Canvas 2D, vanilla JS ES modules, vitest for tests.

```js
  // Background refresh вҖ” updates list without showing loading spinner
  _silentRefresh() {
    let friendData = null
    let myKV       = null
    let done       = 0

    const tryBuild = () => {
      done++
      if (done < 2) return
      // Build new list but don't flash loading state
      const prevList = this._rankList
      this._build(friendData || [], myKV)
      // _build sets this.loading = false and this._rankList = newList
      // also update the cache
      this.game._rankCache = { list: this._rankList, ts: Date.now() }
    }

    cloud.getFriendKV([RANK_KEY],
      (data) => { friendData = data || []; tryBuild() },
      ()     => { friendData = [];         tryBuild() }
    )
    cloud.getMyKV([RANK_KEY],
      (kvList) => {
        const kv = (kvList || []).find(k => k.key === RANK_KEY)
        myKV = kv ? (parseInt(kv.value, 10) || 0) : 0
        tryBuild()
      },
      () => { myKV = null; tryBuild() }
    )
  }
```
